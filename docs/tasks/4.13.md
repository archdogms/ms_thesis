### 四、本阶段具体工作任务

#### 任务1：知识图谱实体的空间定位（知识地图核心）

**目标**：为知识图谱中的地名/建筑遗迹实体匹配经纬度坐标。

**操作**：
1. 从 `entities.json` 筛选 `type="地名"` 的 1,148 个实体和 `type="建筑遗迹"` 的 1,304 个实体
2. 与 `poi_cleaned.json` 的 POI 名称做匹配（名称包含匹配），获取坐标
3. 与 `cultural_anchors.json` 的锚点名称做匹配，补充坐标
4. 统计匹配率，评估空间覆盖情况

**输出**：一个带坐标的地点实体表（如 `output/tables/located_entities.csv`）

#### 任务2：以地点为轴心的关系反向索引（知识地图核心）

**目标**：为每个已定位的地点实体，汇总所有与之关联的实体和关系。

**操作**：
1. 从 `relations.json` 中提取所有 `位于`、`活动于`、`出生于`、`创建修建`、`始建于`、`承载文化` 等空间语义关系
2. 以 target（地点实体）为键，反向聚合 source（人物/非遗/事件等）
3. 形成"地点 → 关联实体列表 → 关联关系类型 → 提及频次"的索引结构

**输出**：一个地点中心的关联索引（如 `output/tables/place_entity_index.json`）

#### 任务3：评论数据的空间挂载（认知地图核心）

**目标**：将评论统计结果（条数、评分、情感分布）关联到POI坐标上。

**操作**：
1. 读取 `review_summary_merged.json` 和 `review_poi_matched.csv`
2. 通过匹配关系，将每个有评论的景点对应到 `poi_cleaned.json` 中的 POI
3. 为匹配到的 POI 附加评论字段（评论数、平均评分、正/中/负面比例）

**输出**：一个带评论属性的POI表（如 `output/tables/poi_with_reviews.json`）

#### 任务4：三图统一数据结构设计

**目标**：设计以 POI/地点 为核心的统一数据 schema，三张地图共享一个库，以属性区分。

**操作**：设计一个统一的 JSON/数据库结构，每个地点记录包含：
```
{
  "id": "POI_xxx",
  "name": "xxx",
  "lng": 113.xx,
  "lat": 23.xx,
  "town": "xx镇",
  "layers": {
    "tourism": { ... },      // 旅游点地图层：类别、评分、POI属性
    "cognition": { ... },    // 认知地图层：评论统计、情感分布
    "knowledge": { ... }     // 知识地图层：关联实体、关系、提及频次
  }
}
```

**输出**：统一的三图数据文件（如 `output/tables/triple_map_data.json`）

#### 任务5：可行性验证与缺口报告

**目标**：量化评估三图数据的覆盖率和缺口。

**需要回答的关键问题**：
- 知识图谱中多少地名实体能成功匹配到坐标？（匹配率）
- 有评论的403个景点中，多少能匹配到POI？（覆盖率）
- 三图在镇街维度的覆盖是否均衡？
- 非遗91项中，有多少能通过地点桥梁同时出现在三张地图上？

**输出**：可行性评估报告（数字 + 缺口清单）

#### 任务6：三张地图的 GIS 输出与简图

**目标**：将三图数据导出为 ArcGIS 可直接加载的矢量文件，并用 Python 生成简图供快速验证。

**操作**：
1. 将 `triple_map_data.json` 转为三套 GIS 文件（每张地图一套），放入 `output/gis/`：
   - **旅游点地图**：`output/gis/map_tourism.shp`（+ .geojson）— 全量 POI 点位，属性含类别、评分、非遗匹配等
   - **旅游认知地图**：`output/gis/map_cognition.shp`（+ .geojson）— 有评论的 POI 子集，属性含评论数、评分、情感比例
   - **知识地图**：`output/gis/map_knowledge.shp`（+ .geojson）— 已定位的知识图谱地点，属性含提及频次、关联实体数
2. 每套 Shapefile 使用 **WGS84 (EPSG:4326)** 坐标系，字段名用英文缩写以兼容 .dbf 10字符限制
3. 用 Python（geopandas + matplotlib）为每张地图各出一张简图 PNG，叠加镇街边界底图，放入 `output/figures/`

**输出**：
- 3 套 Shapefile + GeoJSON（ArcGIS 可直接打开，做专业制图）
- 3 张简图 PNG（快速验证 + 论文草稿插图备用）

### 六、三张地图的生成方式与数据处理匹配流程（已完成版）

#### 6.1 总体方法：一个统一库 + 三个主题图层

三张地图不是三套孤立数据，而是先构建一个统一主数据文件 `output/tables/triple_map_data.json`，再按图层拆分输出：

- 旅游点地图（tourism layer）
- 旅游认知地图（cognition layer）
- 知识地图（knowledge layer）

最后从同一数据源导出 ArcGIS 可读的 `Shapefile + GeoJSON`。

---

#### 6.2 旅游点地图（空间实践）如何得到

**输入数据**：
- `data/poi/poi_cleaned.json`（13,512 条 POI，含经纬度）
- 辅助字段：`category`、`rating`、`nonheritage_match`、`cultural_anchors`

**处理方式**：
1. 直接以 POI 为点位底座（`lng`/`lat`）；
2. 保留旅游供给属性（类别、评分、是否非遗关联等）；
3. 无需复杂匹配，主要是字段标准化与图层封装。

**结果图层**：
- `output/gis/map_tourism.shp`
- `output/gis/map_tourism.geojson`

---

#### 6.3 旅游认知地图（表征性空间）如何得到

**输入数据**：
- `data/reviews/review_summary_merged.json`（评论汇总）
- `output/tables/review_poi_link.csv`（评论景点名与 POI 的匹配关系）

**匹配逻辑**：
1. 以 `spot_name -> poi_id` 为主键进行挂载；
2. 找不到完全同名时，采用包含匹配补充；
3. 将评论统计映射到 POI 坐标点上。

**核心计算字段**：
- `review_count`、`avg_rating`
- `positive_count`、`neutral_count`、`negative_count`
- `positive_rate`、`negative_rate`
- `sources`

**结果图层**：
- `output/gis/map_cognition.shp`
- `output/gis/map_cognition.geojson`

---

#### 6.4 知识地图（空间表征）如何得到

**输入数据**：
- `data/entities/entities.json`（实体）
- `data/entities/relations.json`（关系）
- `data/anchors/cultural_anchors.json`（文化锚点坐标）
- `data/poi/poi_cleaned.json`（POI 坐标）

**步骤 A：实体空间定位（把“知识实体”落到地图坐标）**
1. 从实体中筛选 `地名` + `建筑遗迹`；
2. 先精确匹配 POI/锚点名称；
3. 再做包含匹配；
4. 过滤无效坐标（如 `0,0`），保留有效点位。

**步骤 B：关系反向索引（把“地点”变成知识中心）**
1. 提取空间相关关系：`位于`、`活动于`、`出生于`、`创建修建`、`始建于`、`承载文化` 等；
2. 以地点实体为 key，聚合相关人物/事件/技艺；
3. 形成每个地点的 `related_entities` 与 `total_related` 指标。

**结果图层**：
- `output/gis/map_knowledge.shp`
- `output/gis/map_knowledge.geojson`

---

#### 6.5 三图统一数据结构是如何拼装的

以 `POI id` 为统一主键，将三类信息装入同一记录：

- `layers.tourism`：旅游供给属性
- `layers.cognition`：评论认知属性（无则为 `null`）
- `layers.knowledge`：知识关联属性（无则为 `null`）

因此每个地点都可以判断属于：
- 仅旅游层；
- 旅游+认知；
- 旅游+知识；
- 三层齐全（高价值样本点）。

---

#### 6.6 从统一库到 ArcGIS 图层的导出方式

由脚本 `code/analysis/build_triple_map.py` 自动完成：

1. 生成中间成果  
   - `located_entities.csv`  
   - `place_entity_index.json`  
   - `poi_with_reviews.json`  
   - `triple_map_data.json`  
   - `triple_map_feasibility.md`
2. 生成 GIS 图层  
   - `map_tourism.shp/.geojson`  
   - `map_cognition.shp/.geojson`  
   - `map_knowledge.shp/.geojson`
3. 生成快速预览图  
   - `output/figures/map_tourism_preview.png`  
   - `output/figures/map_cognition_preview.png`  
   - `output/figures/map_knowledge_preview.png`

坐标系统一为 **WGS84 (EPSG:4326)**，ArcGIS 可直接加载。
