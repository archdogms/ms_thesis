# 本科毕业设计答辩 PPT 结构计划（设计增强版）

题目：基于多源数据的佛山市南海区旅游景点文旅融合潜力研究  
建议页数：18 页  
建议时长：8 分钟  
视觉方向：参考“方案二：地图数据型学术汇报”，以清华紫为主色，辅以地图蓝灰、浅紫、灰绿和少量金色；减少重复卡片，改为地图主视觉、数据大数字、流程河流、左右证据对照、四象限诊断、案例专题和结论墙等多种版式。

## 一、答辩叙事主线

本次答辩不照搬论文目录，而按“研究问题如何被一步步收敛”的方式讲：

1. 开题阶段：提出“文化谱系—景点谱系”的双谱系对比，希望识别文化资源与旅游体验之间的关联、错位和缺失。
2. 中期阶段：完成典籍 OCR、POI 清洗、评论匹配和知识图谱构建，把双谱系设想推进为可计算的数据框架。
3. 定稿阶段：将研究对象收敛为 165 条景点级文旅载体，并构建 CMI、OAI、THI、MI 四个指数，形成可落图、可复核、可转译的文旅融合潜力诊断。

一句话主线：

南海并不缺文化，也不缺旅游空间；真正需要识别的是文化记忆、空间载体和游客体验在哪里接上、在哪里脱节，以及哪些点最值得优先深化。

## 二、设计原则

1. 每页只讲一个判断，但证据更充分：标题写结论，副标题解释边界，主视觉承载数据。
2. 减少“框里放文字”的重复页面：每 2 到 3 页更换一次版式节奏。
3. 图表优先：能够用地图、条形图、矩阵、流程或时间线表达的，不使用大段文字。
4. 口头表达优先：PPT 不是论文搬运，页面文字应帮助老师快速抓住“我为什么这么判断”。
5. 数据真实：所有数字来自论文正文、结果表或已生成图表，不新增未经验证的数据。

## 五、可直接使用的口头主线

开题时，我的核心设想是建立“文化谱系”和“景点谱系”的对比框架，去看文化资源和现实景点之间是否接得上。中期阶段，我完成了典籍 OCR、POI 清洗、评论匹配和知识图谱构建，把这个框架推进到了可计算的层面。最终论文进一步把研究对象收敛到 165 条景点级文旅载体，并用 CMI、OAI、THI 和 MI 四个指数识别南海区文旅融合中的沉睡潜力、空心景点和优先深化对象。

## 七、设计改进要求

1. 不再连续使用同一种矩形卡片版式。
2. 每页至少有一种主视觉：地图、流程、图表、矩阵、时间线或大数字。
3. 正文文字比上一版更翔实，但仍保持答辩可讲，不堆论文原文。
4. 页眉页脚保持统一，但正文区允许不同构图。
5. 所有最终 PPT 页面仍可做成整页图片，确保正文、图表和标题不再可编辑。
