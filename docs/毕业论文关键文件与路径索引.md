## B. `output/tables/`：论文图表与正文引用首选

以下文件为**已整理表格**，适合直接做描述性统计、制图或附录。

| 文件 | 数据重点 |
|------|----------|
| **`poi_cleaned.csv`** | 清洗后 POI 全量（镇街、类别、坐标、评分、非遗关键词匹配、文化锚点挂接等）。**首选**。 |
| **`poi_llm_cleaned.csv`** | LLM 辅助纠类 / 文旅相关性后的 POI 表（若正文写「本地大模型清洗」）。 |
| **`review_summary_merged.csv`** | 多平台评论汇总到景点粒度（条数、来源等）。 |
| **`reviews_detail.csv`** | 评论明细（需细到单条评论时）。 |
| **`review_poi_matched.csv`** | 评论侧景点名与 POI 匹配结果（五段式匹配产出）。 |
| **`review_poi_link.csv`** | 评论与 POI 的链接关系表。 |
| **`nonheritage_full90.csv`** | 南海非遗名录扩展表（耦合分析主输入之一）。 |
| **`nonheritage.csv`** | 名录精简/中间版（若与 full90 并存，以论文实际采用版本为准）。 |
| **`coupling_analysis.csv`** | 非遗—图谱—POI 耦合明细。 |
| **`coupling_summary.json`** | 耦合汇总统计。 |
| **`nh_coupling_summary.csv`** | 南海耦合分项汇总。 |
| **`nh_coupling_detail.json`** | 南海耦合详细结果（JSON）。 |
| **`nh_entity_match.csv`** | 实体侧匹配（与图谱 NER/提及相关）。 |
| **`nh_poi_match.csv`** | POI 侧匹配。 |
| **`culture_entities.csv`** | 文化实体列表（论文「文化谱系」量化）。 |
| **`culture_genealogy_stats.json`** | 文化谱系统计。 |
| **`experience_scores.csv`** | 体验/评分类衍生指标（若分析体验度）。 |
| **`spatial_analysis_results.json`** | 空间分析（KDE 等）结果摘要。 |
| **`spatial_town_stats.csv`** | **镇街尺度**空间/资源对比（答辩「东旅西文」等）。 |

## E. 开题 / 中期叙事要点（精简）

- **开题**：文化谱系 ↔ 景点谱系；文化载体作锚点；OCR / API → 知识图谱 → 对比分析。  
- **中期**：题目强调**旅游景区**与**潜力**；中期答辩时口径为 541 万字语料、13,113 POI、16,391 评论、8,048/19,382 图谱、91 项非遗桥梁、Neo4j、KDE；**文化载体 165 条级**与 POI 邻近关联。  
- **4.15 修订（论文版）**：语料口径更新为约 **899 万字**；POI 清洗迭代为 **13,512 条**；耦合桥梁调整为"物质遗产为主（165 条载体：文物 80 / 名村 12 / 文化景观 19 / 圩市 18 / 非遗锚点 36）、非遗为动态补充"；新增量化指数（CMI、OAI、THI、MI）与相关性潜力条件分析。
- **一致性**：正文数字以 **`output/tables/` 与 `progress.json`** 核对为准；答辩 PPT 若与表不一致，以**最新导出表**更新 PPT。
